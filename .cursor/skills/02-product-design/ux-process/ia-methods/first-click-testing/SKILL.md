# First-Click Testing

## When to Use

- Validate navigation entry points
- Compare alternative structures
- Test first-click decisions before full flow

## Process

1. **Prepare** — Wireframe or screenshot. Task: "Where would you click to [goal]?"
2. **Run** — Participants indicate first click.
3. **Measure** — First-click accuracy. Confusion points.
4. **Analyze** — Correct vs. wrong entry points.

## Output

- First-click accuracy per task
- Confusion points (wrong clicks)
- Entry-point recommendations

Use [synthesis-template.md](synthesis-template.md) for handover format.

## Downstream Use

- Feeds IA revision (entry points, labels)
- Informs wireframe design (primary CTAs)
