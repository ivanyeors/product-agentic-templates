# Content Model

## When to Use

- Content-heavy product
- CMS-driven
- Multiple content types with metadata
- Tagging, search, faceted browsing

## Process

1. **Identify** — Content types from PRD, Content Audit, or Card Sort.
2. **Define** — Fields per type, data types, required/optional.
3. **Relationships** — Parent-child, references between types.
4. **Metadata** — Tags, categories, attributes for retrieval.

## Output

- Content type schema
- Field definitions
- Relationships

## Handover Template

Use [handover-template.md](handover-template.md) for handover format.

## Downstream Use

- Feeds IA Document Content Types table
- Feeds Backend Design schema
- Feeds Taxonomy Design (when tagging)
