# Taxonomy Design

## When to Use

- Complex content, tagging
- Search and filter requirements
- Faceted classification
- Multiple dimensions for browsing

## Process

1. **Terms** — Controlled vocabulary (closed list).
2. **Hierarchy** — Parent-child relationships.
3. **Facets** — Independent dimensions (e.g., type, audience, topic).
4. **Mapping** — Which content types use which terms.

## Output

- Controlled vocabulary
- Term hierarchy
- Facet definitions
- Taxonomy spec for metadata

## Handover Template

Use [handover-template.md](handover-template.md) for handover format.

## Downstream Use

- Feeds IA Document (metadata, tagging)
- Feeds Content Model (metadata attributes)
- Feeds Backend (tag/category schema)
