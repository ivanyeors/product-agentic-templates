# Content Audit

## When to Use

- Existing product or redesign
- Content-heavy IA
- Pre-IA: need to inventory and assess before defining structure

## Process

1. **Inventory** — Catalog all pages, content types, media. Use tools (Screaming Frog, analytics) or manual audit.
2. **Analyze** — Evaluate each item: quality, relevance, accuracy, freshness.
3. **Recommend** — Keep / Revise / Remove / Consolidate. Identify gaps.

## Output

- Content inventory (spreadsheet or table)
- Quality assessment per item
- Recommendations for IA

## Synthesis Template

Use [synthesis-template.md](synthesis-template.md) for handover format.

## Downstream Use

- Feeds IA Document Content Types table
- Informs Site Map Structure
- Content Model (when content-heavy)
