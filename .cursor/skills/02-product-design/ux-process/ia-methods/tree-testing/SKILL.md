# Tree Testing

## When to Use

- Proposed navigation structure exists
- Validate findability before design
- Test before investing in wireframes

## Process

1. **Prepare** — Tree structure (text-only, no visual design). 5–10 tasks.
2. **Run** — 20+ participants.
3. **Measure** — Success rate, directness, time per task.
4. **Analyze** — Where do users go wrong?

## Output

- Success rate per task
- Directness (first-click success)
- Time per task
- Findability report for IA revision

## Synthesis Template

Use [synthesis-template.md](synthesis-template.md) for handover format.

## Downstream Use

- Feeds IA revision (relabel, restructure)
- Informs First-Click Testing (if needed)
