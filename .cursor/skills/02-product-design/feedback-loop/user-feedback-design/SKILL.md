# User Feedback Design

## When to Use

- Feedback Channels Plan includes in-app feedback
- Need to design feedback form, NPS, bug report UI
- Producing UF-xxx and WF-xxx for feedback screens

## Process

1. **Reference** — Feedback Channels Plan (touchpoints, placement).
2. **Flow** — Map feedback submission flow (entry → form → success/error).
3. **Wireframe** — Specify feedback UI (form fields, states, validation).
4. **Trace** — UF-xxx, WF-xxx, Covers: FR-xxx.

## Output

- User flow(s) for feedback (UF-xxx)
- Wireframe specs for feedback UI (WF-xxx)
