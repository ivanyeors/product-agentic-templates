# Feedback Channels Plan

## When to Use

- PRD includes feedback collection (FR-xxx)
- Product needs user feedback, NPS, bug reports
- Planning feedback infrastructure from day one

## Process

1. **Inventory** — Which channels: in-app form, email, support ticket, app store reviews, etc.
2. **Placement** — Where each surfaces in IA (nav, settings, help, modal).
3. **Touchpoints** — When/where users can give feedback (post-task, always-available, etc.).
4. **Document** — Feedback Channels Plan for handover.

## Output

- Feedback Channels Plan document
- Channels, touchpoints, IA placement

## Handover Template

Use [handover-template.md](handover-template.md) for handover format.

## Downstream Use

- Informs IA (utility nav, settings, help)
- Informs user-feedback-design (flows, wireframes)
- Informs Backend Design (endpoints, webhooks)
- Informs Integration (third-party tools)
